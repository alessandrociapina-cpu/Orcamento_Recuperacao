📄 Documentação Técnica e Arquitetural (v16)

O Gerador de Orçamento de Recuperação Estrutural e Umidade foi desenvolvido com base em princípios rigorosos de segurança da informação, operando integralmente como uma aplicação descentralizada (Client-Side e Offline-First). Esta arquitetura foi escolhida especificamente para contornar limitações severas de proxy e firewall presentes em redes corporativas críticas, garantindo a produtividade da engenharia técnica no campo.

A. Processamento de Dados (Zero-Server Architecture)

Diferente de sistemas web convencionais, o aplicativo não requer envio de imagens ou rascunhos para servidores em nuvem. Todo o ciclo de vida dos dados ocorre estritamente dentro da memória RAM local do dispositivo do utilizador.

B. Prevenção de Perda de Dados em Campo (IndexedDB)

O aplicativo embute um módulo assíncrono de persistência contínua (Auto-Save). A cada interação, a estrutura inteira de dados do utilizador e as imagens em Base64 são "congeladas" no banco de dados NoSQL do próprio navegador (IndexedDB).

C. Sincronização Automática SINPLAN e Offline

A aplicação atua como um PWA (Progressive Web App) governada por um Service Worker. No primeiro acesso com ligação à internet, o sistema realiza o fetch do banco de preços SINPLAN embutido no repositório (via JSON UTF-8) e armazena na memória persistente.
